# ToDoLab UI/UX Backlog

기능 개발과 분리해서 현재 화면의 사용성, 피드백, 접근성을 개선한다.
각 항목은 가능한 한 하나의 독립 커밋으로 진행한다.

참고한 제품:

- Todoist: Today, Overdue, 필터와 빠른 일정 변경
- Microsoft To Do: My Day, Suggestions, 하루 단위 집중 화면
- Things: Today, Upcoming, Inbox, 실행 순서 중심 목록
- TickTick: Smart List, Suggested Tasks, Task Detail, Calendar View

## 진행 원칙

- 기능 요구가 필요한 항목은 `ROADMAP.md`로 이동한다.
- UI 수정은 서버 모델 변경 없이 해결 가능한 범위를 우선한다.
- 한 커밋에서 여러 화면을 동시에 대규모로 바꾸지 않는다.
- 각 변경 후 주요 화면 수동 확인과 전체 테스트를 실행한다.
