# ToDoLab Roadmap

ToDoLab은 할 일을 많이 쌓아두는 앱이 아니라, 사용자가 오늘 실제로 끝낼 수 있는 크기로 일을 고르고 실행하게 돕는 앱이다.

핵심 흐름은 아래와 같다.

```text
빠르게 기록
-> 오늘 할 일로 선별
-> 실행 가능한 크기로 정리
-> 완료 처리
-> 완료 로그로 성취감 확인
-> 미룬 일은 다시 판단
```

## 1. 제품 원칙

- 기록은 빠르게 한다.
- 오늘 할 일은 너무 많이 쌓지 않는다.
- 일정과 할 일은 사용자가 헷갈리지 않게 보여준다.
- 큰 일은 작은 실행 단위로 쪼갤 수 있어야 한다.
- 완료한 일은 눈에 보이게 남긴다.
- 계속 미루는 일은 방치하지 않고 다시 정리하게 한다.
- 기능은 한 번에 크게 붙이지 않고, 데이터 모델, API, 화면 순서로 작게 붙인다.

## 2. 현재 단계

현재 단계는 **1차 MVP 완료, 2차 핵심 고도화와 UI/UX 정리 완료 후 다음 기능 고도화 준비**다.

완료된 핵심 흐름:

- 빠른 기록
- 기록함
- Today 실행 목록
- 완료 처리
- 완료 로그
- 미완료 이월
- 지난 미완료 정리
- D-Day와 Today 할 일 연결
- Today 추천 작업
- Today 실행 순서 정리
- Today 실행일과 Calendar 일정 동기화
- 미룬 이유 선택
- 전반 UI/UX 정리

현재 앱의 중심은 `Today`다.
Calendar는 날짜 맥락을 확인하는 보조 화면이고, D-Day는 목표와 오늘 실행을 연결하는 차별화 화면이다.

## 3. 지금 가능한 사용자 흐름

### 빠르게 기록하기

사용자는 제목만 입력해서 할 일을 기록할 수 있다.
날짜가 없으면 기록함에 들어가고, 나중에 오늘 할 일로 고를 수 있다.

현재 상태:

- [x] 제목만으로 빠른 기록
- [x] 날짜 없는 항목은 기록함으로 저장
- [x] Today 안에서 기록함 항목 확인
- [x] 기록함 항목을 오늘 할 일로 이동

남은 과제:

- [ ] `내일 3시 병원 예약` 같은 날짜/시간 포함 빠른 입력
- [ ] 파싱 결과를 저장 전 확인하는 UI

### 오늘 할 일 실행하기

사용자는 Today에서 오늘 할 일을 보고, 순서를 정하고, 완료하거나 내일로 넘길 수 있다.

현재 상태:

- [x] Today 할 일 목록
- [x] 캘린더 일정과 실행 할 일 섹션 분리
- [x] 완료 처리
- [x] 내일로 이월
- [x] 기록함으로 이동
- [x] 드래그 앤 드랍 순서 변경
- [x] 완료한 일 하단 표시

남은 과제:

- [ ] 오늘 할 일이 너무 많을 때 안내
- [ ] 아침 계획 또는 하루 시작 안내

### 지난 미완료 정리하기

어제 이전에 하기로 했지만 완료하지 못한 일은 Today 진입 시 별도 섹션으로 보여준다.
자동으로 오늘 목록에 넣지 않고, 사용자가 직접 다시 판단하게 한다.

현재 상태:

- [x] 지난 미완료 조회
- [x] Today 상단에 지난 미완료 섹션 표시
- [x] 오늘 하기
- [x] 내일 하기
- [x] 날짜 다시 정하기
- [x] 기록함으로 이동
- [x] 완료 처리
- [x] 삭제
- [x] 이월 횟수 표시
- [x] 3회 이상 이월 시 미룬 이유 선택

남은 과제:

- [ ] 미룬 이유에 따른 후속 조치
- [ ] 작게 쪼개기
- [ ] 하지 않기로 정리
- [ ] 보류함 또는 나중에 할 일 검토

### D-Day 목표 관리하기

D-Day는 단순 카운트다운이 아니라, 목표까지 가까워지기 위한 오늘 할 일을 연결하는 기능이다.

현재 상태:

- [x] D-Day 목표 등록
- [x] 목표 날짜와 남은 날짜 표시
- [x] D-Day와 Today 할 일 연결
- [x] D-Day에서 연결된 할 일 확인
- [x] Calendar에 D-Day 표시

남은 과제:

- [ ] 목표별 진행률
- [ ] D-Day 임박 알림
- [ ] 목표별 주간 실행 요약

### Calendar로 날짜 맥락 보기

Calendar는 일정과 날짜별 실행 흐름을 보는 화면이다.
Today와 Calendar가 다른 날짜를 보여주지 않도록 날짜 규칙을 단순화했다.

현재 상태:

- [x] 주간 Calendar
- [x] 월간 Calendar
- [x] 날짜별 할 일 표시
- [x] 날짜별 완료 로그 표시
- [x] D-Day 점 표시
- [x] 일정, 완료, 미룸, D-Day 범례와 필터
- [x] Today 실행일과 Calendar 일정 동기화

남은 과제:

- [ ] 향후 7일 실행 계획
- [ ] 날짜별 과부하 안내
- [ ] Calendar에서 더 빠른 날짜 재조정

## 4. 다음 작업 우선순위

다음 작업은 아래 순서가 좋다.

### 1. 빠른 입력 고도화

사용자가 기록하는 순간에 날짜와 시간을 함께 넣을 수 있게 한다.
Todo 앱에서 입력 속도는 체감 품질에 직접 영향을 준다.

예시:

- `내일 3시 병원 예약`
- `금요일 포트폴리오 제출`
- `6월 30일 종일 세금 신고`

구현 순서:

1. [ ] 빠른 입력 파서 문법 정리
2. [ ] 오늘, 내일, 요일, `M월 D일`, `HH시` 파싱
3. [ ] 파싱 결과 미리보기 UI
4. [ ] 확정 시 날짜와 시간 저장
5. [ ] 파싱 실패 시 기존처럼 기록함 저장

완료 기준:

- 날짜 없는 입력은 기존처럼 기록함에 저장된다.
- 날짜가 있는 입력은 Today와 Calendar에 같은 날짜로 보인다.
- 시간 입력은 Calendar에서 시간 일정처럼 보인다.
- 사용자는 저장 전에 파싱 결과를 이해할 수 있다.

### 2. 검색과 필터

할 일이 쌓이면 Today만으로는 오래된 기록이나 특정 목표의 일을 찾기 어렵다.
처음에는 단순 검색과 자주 쓰는 필터부터 시작한다.

초기 필터:

- 기록함
- 오늘
- 지난 미완료
- 완료
- D-Day 연결됨
- 3회 이상 이월
- 날짜 없음

구현 순서:

1. [ ] 제목/설명 검색 API
2. [ ] 상태, 날짜 있음/없음, D-Day 연결 여부 필터
3. [ ] More 또는 별도 검색 화면
4. [ ] 자주 쓰는 필터 칩
5. [ ] 필요하면 저장된 필터로 확장

완료 기준:

- 사용자는 오래된 기록을 제목으로 찾을 수 있다.
- 미룬 일, 날짜 없는 일, D-Day 연결 일을 빠르게 볼 수 있다.
- 필터는 Today 흐름을 방해하지 않고 보조 화면에서 제공된다.

### 3. 향후 7일 실행 계획

오늘 이후 일주일 동안 실행하기로 한 일을 날짜별로 미리 본다.
특정 날짜에 할 일이 몰리는 문제를 줄이는 기능이다.

구현 순서:

1. [ ] 오늘 이후 7일의 실행 할 일 조회
2. [ ] 날짜별 개수와 목록 표시
3. [ ] 날짜 간 이동 또는 날짜 재설정
4. [ ] Calendar 일정과 TODO 실행 계획을 구분해 표시

완료 기준:

- 사용자는 앞으로 7일의 할 일 분포를 볼 수 있다.
- 특정 날짜에 일이 많은지 바로 알 수 있다.
- 날짜 변경 후 Today와 Calendar가 같은 날짜를 보여준다.

### 4. 큰 할 일 쪼개기

큰 할 일은 미루기 쉽다.
처음에는 AI 없이 사용자가 직접 하위 작업을 추가하는 방식으로 시작한다.

구현 순서:

1. [ ] 상위/하위 작업 관계 설계
2. [ ] 상세 모달에서 하위 작업 추가
3. [ ] 하위 작업 완료 체크
4. [ ] 상위 작업 진행률 표시
5. [ ] 큰 작업을 Today에 넣을 때 쪼개기 제안

완료 기준:

- 큰 작업 하나에 여러 하위 작업을 붙일 수 있다.
- 하위 작업 완료율이 보인다.
- 상위 작업은 하위 작업 상태와 모순되지 않는다.

### 5. 미룬 일 후속 조치

지금은 3회 이상 이월된 일을 감지하고 이유를 저장한다.
다음 단계는 이유에 따라 사용자가 바로 행동할 수 있게 만드는 것이다.

후속 액션 후보:

- 작게 쪼개기
- 보류함으로 이동
- 날짜 다시 정하기
- 삭제
- 하지 않기로 정리

완료 기준:

- 3회 이상 이월된 일은 별도 상태로 눈에 띈다.
- 이유 선택 후 다음 행동을 제안한다.
- 삭제와 `하지 않기로 정리`는 구분한다.

## 5. 이후 고도화 후보

### 오늘 할 일 개수 제한

오늘 할 일이 너무 많아지면 실행력이 떨어진다.
강제로 막기보다 권장 개수를 넘었을 때 안내한다.

후보:

- 기본 권장 개수 3개 또는 5개
- 초과 시 안내 문구
- 오늘 할 일 요약에서 과부하 표시

### 완료와 연속 실행 요약

과한 게임화보다 조용한 성취감을 목표로 한다.

후보:

- 오늘 완료 개수
- 이번 주 완료 개수
- 연속 실행 일수
- 어제보다 나아진 점

### 주간 완료 리포트

완료 로그와 이월 데이터를 기반으로 한 주를 정리한다.

후보:

- 이번 주 완료한 일 개수
- 카테고리별 완료 개수
- 가장 많이 처리한 카테고리
- 계속 미룬 일 목록
- 다음 주로 넘길 일 목록

### 저녁 회고

밤에 앱을 열었을 때 하루를 정리할 수 있게 한다.

후보:

- 오늘 완료한 일
- 오늘 못 한 일
- 내일로 넘길 일
- 계속 미루는 일

### AI 쪼개기

직접 하위 작업을 추가하는 흐름이 안정화된 뒤 붙인다.

예시:

- 사용자가 `이직 준비하기`를 입력한다.
- AI가 작은 실행 단위 후보를 제안한다.
- 사용자가 선택한 항목만 Today 또는 하위 작업으로 만든다.

## 7. 개발 참고

기능을 이어서 개발하거나 기존 동작을 확인할 때 우선 살펴볼 위치다.
제품 흐름은 앞쪽 로드맵을 기준으로 보고, 구체적인 구현은 이 섹션의 파일들을 기준으로 확인한다.

### 핵심 도메인

- `src/main/java/com/todolab/task/domain/Task.java`
- `src/main/java/com/todolab/task/domain/TaskStatus.java`
- `src/main/java/com/todolab/task/domain/TaskType.java`
- `src/main/java/com/todolab/task/domain/DeferReason.java`
- `src/main/java/com/todolab/dday/domain/DdayGoal.java`

현재 중요한 Task 개념:

- `INBOX`: 날짜 없이 기록된 항목
- `TODAY`: 특정 날짜에 실행하기로 한 항목
- `DONE`: 완료된 항목
- `targetDate`: 사용자가 실행하기로 한 날짜
- `startAt/endAt`: 시간이 필요한 경우의 일정 시간
- `todayOrder`: Today 안에서 사용자가 정한 실행 순서
- `carryOverCount`: 이월 횟수
- `deferReason`: 미룬 이유

날짜 규칙:

- 날짜가 없으면 기록함이다.
- 오늘, 내일 또는 특정 날짜로 보내면 `targetDate`가 생긴다.
- 날짜가 있는 TODO는 Calendar에도 같은 날짜로 보인다.
- 시간이 없으면 종일 할 일처럼 보인다.
- 기록함으로 이동하면 날짜와 시간이 제거된다.

## 8. 개발 원칙

- 사용자가 매일 들어왔을 때 Today에서 바로 실행할 수 있어야 한다.
- 기능 요구가 불명확하면 Today 흐름을 우선한다.
- 기존 구조를 한 번에 갈아엎지 않는다.
- 데이터 모델 변경은 API, 화면, 테스트까지 함께 닫는다.
- AI 기능은 핵심 실행 흐름이 안정된 뒤 붙인다.
- 완료 로그와 이월 데이터는 리포트와 미룬 일 관리의 기반으로 본다.

## 9. 인프라 후보

지금 당장 Redis나 Kafka를 붙이지 않는다.
명확한 병목이나 비동기 요구가 생겼을 때 도입한다.

Redis 후보:

- 로그인 이후 세션 저장소
- Today 조회 캐시
- Done Log 집계 캐시
- 자연어 파싱 결과 임시 저장
- rate limit

Kafka 후보:

- Task 생성/완료/이월 이벤트 발행
- 완료 로그 기반 주간 리포트 생성
- 일정 공유 알림 이벤트 처리
- 자연어 일정 등록 요청 비동기 처리
- 일일 요약 메일 또는 저녁 회고 알림 처리
