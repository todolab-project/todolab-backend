# UI/UX Responsive Checklist

모바일과 데스크톱 화면 수정 후 같은 기준으로 회귀를 확인하기 위한 점검표다.
화면 폭은 실제 기기와 브라우저 개발자 도구에서 자주 쓰는 기준을 함께 둔다.

## 기준 뷰포트

- 320px: 가장 좁은 모바일 기준. 버튼, 날짜, 배지가 줄바꿈되어도 가로 스크롤이 없어야 한다.
- 375px: 일반 소형 모바일 기준. 주요 CTA가 한눈에 보여야 한다.
- 430px: 큰 모바일 기준. 카드 간격과 하단 내비게이션이 답답하지 않아야 한다.
- 768px: 태블릿/작은 데스크톱 진입 기준. 모바일 전용 줄바꿈이 과하게 남지 않아야 한다.
- 1280px: 데스크톱 기준. 본문 폭, 카드 폭, 모달 폭이 너무 넓게 퍼지지 않아야 한다.

## 공통 통과 기준

- 페이지 전체에 가로 스크롤이 생기지 않는다.
- 스크롤은 메인 문서 스크롤 하나를 우선 사용한다.
- 하단 내비게이션이 마지막 카드, 빈 화면 CTA, 모달 버튼을 가리지 않는다.
- 완료, 이월, 저장, 삭제, 기록함 이동 버튼은 최소 44px 터치 영역을 유지한다.
- 긴 제목과 설명은 카드 너비 안에서 자연스럽게 줄바꿈되거나 말줄임된다.
- `종일`, 날짜, D-Day, 완료 상태 배지는 같은 카드 안에서 정보 우선순위가 깨지지 않는다.
- 로딩 중 버튼은 텍스트가 바뀌어도 카드 폭을 과하게 흔들지 않는다.
- 키보드 포커스가 카드, 버튼, 모달 닫기에서 보인다.

## 화면별 확인

### Today

- 빠른 추가 입력창과 추가 버튼이 320px에서도 한 줄로 유지되거나 자연스럽게 축소된다.
- 캘린더 일정과 오늘 할 일 섹션이 분리되어 보인다.
- 미완료/이월 카드에서 이유 선택, 내일로, 기록함 이동 액션이 가로로 넘치지 않는다.
- 드래그 정렬 중 텍스트 선택이 발생하지 않고 카드 위치 피드백이 보인다.
- 빈 화면에서는 다음 행동 버튼이 하단 내비게이션에 가리지 않는다.

### Calendar

- 주간/월간 전환, 범례, 필터 버튼이 320px에서 줄바꿈되어도 읽을 수 있다.
- 월간 그리드의 날짜 숫자와 점 표시가 겹치지 않는다.
- 날짜 선택 후 목록 카드가 화면 아래에서 잘리지 않는다.
- 과거 날짜, 오늘, 선택 날짜 스타일이 서로 구분된다.

### Inbox

- `오늘 할 일로`, `내일로`, 삭제 등 우측 액션이 좁은 화면에서 카드 밖으로 밀리지 않는다.
- 날짜가 없는 기록함 할 일은 상태 안내가 과하게 길지 않다.
- 빈 화면 문구와 빠른 추가 동선이 같은 톤으로 보인다.

### D-Day

- 목표 추가 폼은 360px 이하에서 한 칸 레이아웃으로 전환되어 입력창과 날짜가 눌리지 않는다.
- 목표 카드의 D-Day 값, 연결된 할 일 개수, 삭제 버튼이 한 화면 폭 안에 들어온다.
- 목표를 펼쳤을 때 연결된 할 일 목록과 액션 버튼이 겹치지 않는다.

### Task Detail Modal

- 제목, 일정, 설명 영역의 라벨 톤과 아이콘 사용이 일관된다.
- 일정 편집에서 같은 날/여러 날 전환 시 비활성 영역이 명확하다.
- 저장, 기록함 이동, 삭제 버튼이 모달 하단에서 가려지지 않는다.
- 320px에서 모달 내용이 세로 스크롤로 접근 가능하고 배경 페이지가 함께 스크롤되지 않는다.

## 변경 후 운영

1. UI 변경 커밋 전 `320px`, `375px`, `430px`, `1280px`를 최소 기준으로 확인한다.
2. 입력, 완료, 이월, 기록함 이동, 삭제, 모달 저장까지 한 번씩 수행한다.
3. 깨지는 화면이 있으면 CSS 보정과 함께 이 문서의 화면별 확인 항목도 갱신한다.
