# ToDoLab Backend 작업 규칙

이 저장소에서는 ToDoLab 백엔드만 작업한다.

## 작업 범위

- 기본 작업 대상은 Spring Boot 백엔드 코드, 백엔드 테스트, 백엔드 설정, Dockerfile, Gradle 설정이다.
- 모바일 앱 구현, 모바일 UI, Expo 설정, 모바일 패키지 작업은 별도 `todolab-mobile` 저장소에서 진행한다.
- 백엔드 API 변경이 모바일에 영향을 줄 수 있으면 코드 변경은 백엔드에 한정하되, 영향 범위는 완료 보고에 적는다.

## 검증

- 백엔드 변경 후에는 가능한 한 `./gradlew test`를 실행한다.
- 모바일 lint, TypeScript 검사, Expo 실행은 이 저장소의 검증 범위에 포함하지 않는다.

## 변경 관리

- 이 저장소에는 모바일 소스와 모바일 전용 문서를 추가하지 않는다.
- 백엔드와 모바일 변경은 각각의 저장소에서 별도의 이슈, 브랜치, 커밋, PR로 관리한다.
- API 계약이 바뀌면 모바일 저장소에 필요한 후속 작업을 별도로 기록한다.

## 브랜치 운영

- `main`은 모바일 개발이 기준으로 삼는 안정 브랜치로 유지한다.
- 백엔드 변경은 직접 `main`에 커밋하지 않고 작업 브랜치에서 진행한다.
- 브랜치 이름은 짧게 `feat/back/<topic>`, `fix/back/<topic>`, `chore/back/<topic>` 형식을 사용한다.
- API URL, 요청/응답 DTO, 오류 코드 변경은 모바일에 영향을 주는 breaking change인지 확인한다.
- breaking change는 가능한 한 기존 API와 새 API를 함께 유지하거나, 모바일 반영 계획을 먼저 정리한 뒤 진행한다.
- `main` 반영 전에는 `./gradlew test`를 통과시킨다.
