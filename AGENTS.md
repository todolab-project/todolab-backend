# ToDoLab Backend 작업 규칙

이 저장소에서는 ToDoLab 백엔드만 작업한다.

## 작업 범위

- 기본 작업 대상은 Spring Boot 백엔드 코드, 백엔드 테스트, 백엔드 설정, Dockerfile, Gradle 설정이다.
- 서버 렌더링 화면과 정적 리소스는 이 백엔드 애플리케이션이 제공하는 범위 안에서 작업한다.
- 백엔드 API 변경이 외부 클라이언트에 영향을 줄 수 있으면 영향 범위를 완료 보고에 적는다.

## 검증

- 백엔드 변경 후에는 가능한 한 `./gradlew test`를 실행한다.
- 프런트엔드 전용 패키지 매니저나 클라이언트 앱 검사는 이 저장소의 검증 범위에 포함하지 않는다.

## 변경 관리

- 이 저장소에는 백엔드 애플리케이션과 직접 관련 없는 소스나 전용 문서를 추가하지 않는다.
- API 계약이 바뀌면 호환성, 마이그레이션 필요 여부, 영향 범위를 별도로 기록한다.
- 작업은 가능한 한 하나의 논리적 커밋 단위로 진행한다.
- 커밋이나 푸시를 진행하기 전에는 사용자 확인을 받는다.

## 브랜치 운영

- `main`은 배포 가능한 안정 브랜치로 유지한다.
- 백엔드 단독 저장소이므로 필요한 경우 `main`에 직접 커밋할 수 있다.
- 작업 브랜치를 만들 때는 짧게 `feat/<topic>`, `fix/<topic>`, `chore/<topic>` 형식을 사용한다.
- API URL, 요청/응답 DTO, 오류 코드 변경은 breaking change인지 확인한다.
- breaking change는 가능한 한 기존 API와 새 API를 함께 유지하거나, 마이그레이션 계획을 먼저 정리한 뒤 진행한다.
- `main` 반영 전에는 `./gradlew test`를 통과시킨다.
