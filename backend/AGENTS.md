# ToDoLab Backend 작업 규칙

이 디렉터리에서는 ToDoLab 백엔드만 작업한다.

## 작업 범위

- 기본 작업 대상은 Spring Boot 백엔드 코드, 백엔드 테스트, 백엔드 설정, Dockerfile, Gradle 설정이다.
- `../mobile` 앱 구현, 모바일 UI, Expo 설정, 모바일 패키지 작업은 사용자가 명시적으로 요청하지 않는 한 진행하지 않는다.
- `../docs/mobile` 문서는 모바일 작업 요청이 있을 때만 수정한다.
- 백엔드 API 변경이 모바일에 영향을 줄 수 있으면 코드 변경은 백엔드에 한정하되, 영향 범위는 완료 보고에 적는다.

## 검증

- 백엔드 변경 후에는 가능한 한 `./gradlew test`를 실행한다.
- 경로가 루트 기준으로 필요한 경우에도 백엔드 명령은 이 디렉터리에서 실행한다.
- 모바일 lint, TypeScript 검사, Expo 실행은 사용자가 요청한 경우에만 수행한다.

## 변경 관리

- 현재 저장소는 루트 아래 `backend/`와 `mobile/`이 공존하는 구조다.
- 백엔드 작업과 모바일 작업을 같은 변경 단위에 섞지 않는다.
- 커밋이나 PR을 만들 때는 백엔드 변경만 포함되었는지 먼저 확인한다.
